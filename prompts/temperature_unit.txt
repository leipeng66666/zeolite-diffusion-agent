You are an expert in scientific unit conversion for temperature. Given a list of UNIQUE temperature units from a scientific database, determine a precise conversion formula for each unit to convert values to Kelvin (K).

## Target unit: K (Kelvin)

## For each input unit, classify into one of these operations:

### Operation "identity" — already in Kelvin
Output: <unit> → identity | 1.0
- K, kelvin, Kelvin, °K, deg K, deg.K → identity | 1.0
- The value does not change.

### Operation "add" — convert to K by adding a constant
Output: <unit> → add | <offset>
- <offset> is the amount to add: converted_value = original_value + offset
- Examples:
  °C → add | 273.15
  C → add | 273.15
  deg C → add | 273.15
  celsius → add | 273.15
  Celsius → add | 273.15
  °C. → add | 273.15
  deg.C → add | 273.15

### Operation "affine" — convert to K by multiply then add
Output: <unit> → affine | <mul_factor>,<add_factor>
- converted_value = original_value × mul_factor + add_factor
- Examples:
  °F → affine | 0.5555556,255.3722   (K = (°F + 459.67) × 5/9 = °F × 5/9 + 255.3722)
  F → affine | 0.5555556,255.3722
  fahrenheit → affine | 0.5555556,255.3722

### Operation "keep" — NOT a valid temperature, keep as-is
Output: <unit> → keep | N/A
- For null/empty/none/NaN units
- For units that are temperature RANGES (e.g., "273-323 K")
- none → keep | N/A

## CRITICAL RULES:
1. Output ONE line per input unit. No explanations, no extra text.
2. Output format strictly: <original_unit> → <operation> | <factor>
3. Be precise with conversion factors — use at least 6 decimal places.
4. Room temperature / ambient → this is a VALUE, not a unit. The UNIT is still K or °C — classify the unit, not the value.
